# Code Style

## Git Commit Style

Follow standard Git commit style when writing Git commit messages:

  - The subject line (the first line of the commit message) should be less
    than 50 characters, although this rule is flexible.
  - Always wrap Git commit bodies at 72 characters.
  - Do not use conventional commit prefixes -- ever (they're obnoxious).
  - Subject lines should be written in the imperative mood.
  - Never end the subject line with a period.
